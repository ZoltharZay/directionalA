# Falling on Deaf Ears: A Human-AI Comparative Study on Spatial Audio Perception in Virtual Environment

This repository contains the code, datasets, and results for the paper **"Falling on Deaf Ears: A Human-AI Comparative Study on Spatial Audio Perception in Virtual Environment"**. It provides the framework used to evaluate and compare the spatial audio perception capabilities of human subjects against Large Audio Language Models (LALMs).

---

## 📁 Repository Structure

* **`run_evaluation.py`**: The main execution script for running LALM evaluations on the audio datasets.
* **`Sound_3/`**, **`Sound_6/`**, **`Sound_9/`**: Directional sound datasets categorized by audio duration (3, 6, and 9 seconds).
* **`data files/`**: Contains the compiled raw data in CSV format for both human and AI evaluations. It also includes **`BatchSpatialAudioRecorder.cs`**, the Unity C# script used to generate and record the spatial audio dataset.
* **`results-all/`**: Contains the raw, uncompiled output results generated by the Large Audio Language Models.

---

## 🎧 Dataset Details

The audio datasets are split into three main folders based on the duration of the audio clips: `Sound_3` (3 seconds), `Sound_6` (6 seconds), and `Sound_9` (9 seconds). 

Inside each of these duration folders, the audio files are further categorized by spatial complexity:

* **2D Easy**: Left and Right directional audio.
* **2D Mid**: Left, Right, Up, and Down directional audio.
* **3D Easy**: Left, Right, Up, Down, Forwards, and Backwards directional audio.
* **3D Hard**: Dynamic, moving audio trajectories (Right-to-Left, Up-to-Down, Forward-to-Backward). 
    * *Note: The `Sound_9` folder does not include the `3D Hard` subfolder.*

### Dataset Generation
The directional audio clips were recorded within a Unity virtual environment utilizing the Steam Audio plugin to simulate accurate spatialization. The automation code used to batch process, play, and capture these spatialized `.wav` files is available at `data files/BatchSpatialAudioRecorder.cs`.

## 🚀 Running the Evaluation

The model evaluation is handled by `run_evaluation.py`. You can configure the evaluation using various command-line arguments.

### Prerequisites
Ensure you have the necessary dependencies installed (e.g., PyTorch, transformers, or specific model requirements defined in your environment) and that `evaluator_factory.py`, `prompt_templates.py`, `audio_utils.py` and `config.py` are present in your working directory.
-`prompt_templates.py` contains the simple prompts used in the paper and `prompt_templates2.py` contains the spatial prompts used in the paper.
### Basic Usage

To run an evaluation on a specific dataset (e.g., the 3-second sounds), use the following command:

```bash
python run_evaluation.py --model <MODEL_NAME> --data_dir ./Sound_3 --output_dir ./results
```

### Command-Line Arguments

* `--model` **(Required)**: Name of the model to evaluate (must match keys in your `MODEL_CONFIGS`).
* `--data_dir` **(Required)**: Path to the root directory containing the audio data (e.g., `./Sound_6`).
* `--output_dir`: Output directory path for the results (Default: `results`).
* `--iterations`: Number of times to run the evaluation iterations (Default: `1`).
* `--mode`: Evaluation mode to run. Choices include `direct-single` or `all`. (Default: `all`, which runs every mode sequentially).

### Examples

**1. Run all evaluation modes for a generic LALM on the 6-second dataset:**
```bash
python run_evaluation.py --model my-audio-model --data_dir ./Sound_6 --mode all
```

**2. Run a specific mode (`direct-single`) for 3 iterations:**
```bash
python run_evaluation.py --model my-audio-model --data_dir ./Sound_9 --mode direct-single --iterations 3
```
